# Reformed Bytes Website Design Prompt

Use this prompt with Claude, Codex, or another AI design/build assistant to create a top-tier landing page and website foundation for **Reformed Bytes**.

```text
You are a world-class product designer, UX architect, frontend engineer, and brand strategist.

Your work should feel comparable to the websites of:

- Linear
- Stripe
- Vercel
- Apple
- Notion
- Raycast
- Arc Browser
- Anthropic
- Framer
- Clerk

Do not copy these sites. Use them only as inspiration for quality, polish, restraint, typography, motion, and product-level design thinking.

## Objective

Design and build the official website for Reformed Bytes.

This is not a generic personal portfolio.

It is a long-term home for my software projects, engineering experiments, writing, quotes, reading, and reflections.

The website should immediately communicate:

- Craftsmanship
- Intentionality
- Technical excellence
- Thoughtfulness
- Trustworthiness
- Quiet confidence

Visitors should feel like they are looking at the homepage of a modern software company or engineering lab, not a hobby project.

## About Reformed Bytes

Reformed Bytes represents two connected passions:

### Bytes

Modern software engineering:

- Artificial intelligence
- Raspberry Pi
- RF engineering
- SDR/radio projects
- Automation
- Networking
- Infrastructure
- Data visualization
- Self-hosting
- Dashboards
- Thoughtfully designed software

### Reformed

A Reformed Christian worldview influences how I approach my work.

Technology is a gift that can be used creatively, wisely, and for the good of others.

Faith is part of the site's identity, but it should never dominate the experience or alienate visitors who come for the technology.

It should feel authentic, thoughtful, understated, and woven naturally into the brand.

The site should not feel like a church website.

The site should not feel like a theology blog with some tech projects attached.

It should feel like a serious engineering lab whose creator is shaped by Christian conviction.

## Brand Personality

The brand should feel:

- Professional
- Curious
- Humble
- Engineering-focused
- Thoughtful
- Calm
- Trustworthy
- Elegant
- Durable
- Purposeful

Prioritize craftsmanship over hype.

Avoid anything that feels trendy for the sake of being trendy.

## Design Goals

The website should feel:

- Premium
- Calm
- Intentional
- Timeless
- Modern
- Minimal
- Sophisticated
- Readable
- Human

Avoid:

- Loud gradients
- Crypto aesthetics
- Gamer styling
- Excessive animations
- Clutter
- Generic developer portfolio layouts
- Overused tech-bro language
- Cheap glowing cards
- Decorative effects that do not serve the content

## Primary Audience

Primary visitors are people interested in:

- Software engineering
- AI
- RF technology
- Infrastructure
- Raspberry Pi
- Networking
- Automation
- Self-hosting
- Data tools
- Engineering write-ups

Secondary visitors include:

- Friends
- Church members
- People interested in Reformed theology
- Readers interested in quotes, books, and reflections

The site should welcome both audiences without forcing either one to feel out of place.

## Site Philosophy

Design this website as the foundation of an engineering lab, not a static homepage.

It should be capable of expanding into a complete ecosystem of:

- Projects
- Documentation
- Dashboards
- Articles
- Notes
- Quotes
- Reading lists
- Experiments

Every design decision should support long-term growth while maintaining a consistent, premium visual identity.

## Site Structure

Design the website so it can naturally grow over time.

Recommended pages:

- Home
- Projects
- Journal
- Quotes
- Reading
- About

## Home Page

The homepage should include:

- Large hero section
- Brief introduction
- Featured projects
- Current work
- Technology stack
- Latest writing
- Quote of the week
- Optional newsletter section
- Footer

The homepage should immediately communicate what Reformed Bytes is and why it exists.

### Homepage Hero

The hero should immediately communicate purpose.

Possible tone:

```text
Engineering thoughtful software.

Building intelligent systems for AI, RF, automation, and infrastructure.

Technology shaped by craftsmanship and conviction.
```

Include tasteful calls to action such as:

- Explore Projects
- Read Journal
- View Quotes

The hero should feel premium and restrained.

Do not use generic stock imagery.

Do not use a generic developer-at-laptop aesthetic.

## Projects Page

Create a polished project index.

Initially include placeholders for:

- Birmingham FireMap
- Market Monitor
- Sonos Dashboard
- Coffee Dashboard
- Raspberry Pi projects
- SDR Lab
- Future projects

Each project should include:

- Project name
- Short description
- Status
- Technology badges
- Screenshot or visual placeholder
- Link or coming-soon state
- Optional write-up link

Project cards should feel like product cards from a polished SaaS company.

They should not look like GitHub repo tiles.

## Journal Page

Create a modern writing section.

This should feel more like an engineering journal than a traditional blog.

Topics may include:

- AI
- Software architecture
- RF engineering
- Networking
- Coffee
- Raspberry Pi
- Home lab work
- Lessons learned
- Project write-ups
- Reflections on craft, stewardship, humility, and technology

The writing experience should be excellent:

- Beautiful typography
- Good spacing
- Easy reading
- Strong article previews
- Tags/categories
- Date metadata
- Search or filtering if appropriate

## Quotes Page

Create a searchable collection of favorite quotations.

Categories may include:

- Theology
- Leadership
- Engineering
- Wisdom
- Craftsmanship
- Work

Each quote should include:

- Quote text
- Author
- Source, if known
- Tags
- Optional notes

The visual design should make quotations feel elegant and worth reading.

### Quote of the Week

Include a featured quote section on the homepage.

Example:

```text
"There is not one square inch in the whole domain of our human existence over which Christ, who is Sovereign over all, does not cry: Mine."

- Abraham Kuyper
```

Make this elegant and understated.

It should feel like a thoughtful signature of the site, not a sermon.

## Reading Page

Create a page for:

- Currently Reading
- Recently Finished
- Recommended Books
- Future notes

This section can include theology, engineering, biography, history, productivity, and technical books.

The design should feel like a curated bookshelf rather than a generic list.

## About Page

Explain:

- Who I am
- Why Reformed Bytes exists
- My engineering interests
- How faith influences my work
- Why technology, stewardship, creativity, and service matter

Keep the page personal without becoming overly autobiographical.

Possible direction:

```text
Reformed Bytes is my collection of software projects, engineering experiments, and writing.

I work with RF infrastructure and utility communications, enjoy building AI-powered tools and Raspberry Pi projects, and believe technology is one way we exercise creativity and stewardship.

The name Reformed Bytes reflects both my passion for engineering and my conviction that faith shapes how we approach our work, our craft, and our service to others.
```

## Navigation

Keep navigation simple:

- Projects
- Journal
- Quotes
- Reading
- About

Avoid cluttering the navbar.

The navigation should scale as the site grows.

## Visual Direction

Use a premium, dark-mode-first visual system.

The design should use:

- Deep charcoal
- Warm grays
- Off-white text
- Muted accent colors
- Subtle borders
- Soft contrast
- Excellent typography
- Minimal but meaningful motion

Avoid:

- Bright neon
- Rainbow gradients
- Heavy purple/blue SaaS gradients
- Overly beige or brown palettes
- Excessive glassmorphism
- Overdecorated cards

## Typography

Typography should be exceptional.

Use modern font pairings.

Prioritize readability.

The site should feel enjoyable to read for long-form writing.

Consider:

- Strong display type for hero headings
- Highly readable body font
- Thoughtful line length
- Generous but controlled spacing
- Elegant quotation styling
- Distinct but restrained metadata styling

## Components

Create a reusable component system.

Suggested components:

- Site header
- Mobile navigation
- Hero section
- Project card
- Project status badge
- Technology badge
- Article preview card
- Quote card
- Featured quote
- Reading card
- Section heading
- Callout
- Footer
- Tag
- Button
- Card
- Layout wrapper
- MDX article layout

## Technical Requirements

Use a modern stack:

- Next.js
- React
- TypeScript
- Tailwind CSS
- shadcn/ui where useful
- Framer Motion for subtle animation
- Lucide icons
- MDX for writing
- Content collections or a clean content structure
- Cloudflare deployment compatibility

The site must be:

- SEO optimized
- Accessible
- Responsive
- Fast
- Dark mode first
- Lighthouse score above 95
- Component-driven
- Maintainable
- Production-ready

Avoid unnecessary dependencies.

## Motion

Use subtle animations only.

Motion may include:

- Fade
- Slide
- Gentle reveal
- Hover interactions
- Subtle card elevation
- Small transitions between states

Every animation should have a purpose.

Nothing should feel distracting.

The site should feel alive, not flashy.

## Accessibility

Ensure:

- WCAG AA contrast
- Keyboard navigation
- Visible focus states
- Proper semantic HTML
- Good heading structure
- Accessible buttons and links
- Reduced-motion support
- Readable text sizes
- Mobile-friendly tap targets

## Responsive Design

The site should feel excellent on:

- Desktop
- Tablet
- Mobile

Do not simply shrink the desktop layout.

Mobile should feel intentionally designed.

Projects, quotes, and article previews should remain readable and elegant on small screens.

## Content Tone

The writing should sound:

- Clear
- Intelligent
- Calm
- Warm
- Confident but humble
- Technically credible
- Human

Avoid:

- Startup hype
- Overpromising
- Buzzwords
- Corporate filler
- Cheesy Christian branding
- Generic portfolio language
